# 串口通信协议（方案一：ESP32 直连 STM32）

> ⚠️ **必读**：本仓库曾存在多套互相冲突的通信架构，现已统一为「方案一」。
> 旧代码（`car_arm_service.py`、`esp32_cam.ino` 等）属于「PC 中转架构」，已废弃，请勿混用。

---

## 1. 架构概览

```
┌─────────────┐      UART2 (115200)      ┌─────────────┐
│  ESP32-CAM  │  ═══════════════════════► │    STM32    │
│  (视觉识别)  │      0x01 = 齿轮          │  (控制执行)  │
│             │      0x02 = 螺母          │             │
└─────────────┘                           └─────────────┘
       ▲                                        │
       └──────────── 无需 PC ────────────────────┘
```

- **无 PC 参与**：ESP32 本地完成识别后直接通知 STM32，小车可脱机自主运行。
- **单向通信为主**：ESP32 → STM32 发指令，STM32 不回传（如需回传可扩展）。

---

## 2. 物理层

### 2.1 接线定义

| ESP32 / ESP32-CAM | STM32 | 说明 |
|:---|:---|:---|
| **GPIO17 (TX2)** | **PA3 (RX)** | ESP32 发识别结果 |
| **GPIO16 (RX2)** | **PA2 (TX)** | 预留回传通道（当前未使用） |
| **GND** | **GND** | **必须共地** |

### 2.2 串口参数

| 参数 | 值 |
|:---|:---|
| 波特率 | **115200** |
| 数据位 | 8 |
| 停止位 | 1 |
| 校验 | 无 (None) |
| 流控 | 无 |

### 2.3 STM32 端串口

- 使用 **USART2**（PA2/PA3）
- 中断处理函数：`USART2_IRQHandler`（位于 `Hardware/Usart.c`）
- 接收逻辑：单字节匹配 `0x01` 或 `0x02`，直接存入全局变量 `rx_cmd`

---

## 3. 应用层协议

### 3.1 指令定义

| 指令字节 | 含义 | STM32 动作 |
|:---|:---|:---|
| `0x01` | 识别到 **齿轮** | `Loop_Vision_Auto()` 执行齿轮抓取序列 |
| `0x02` | 识别到 **螺母/螺栓** | `Loop_Vision_Auto()` 执行螺母抓取序列 |
| 其他 | 保留 / 无效 | 忽略 |

### 3.2 发送时序

1. ESP32 每 200ms 采集一帧图像并识别
2. 若连续 N 帧（如 3 帧）识别到同一物体，发送对应指令
3. 发送后进入 **1.5s 冷却期**，避免重复触发
4. STM32 收到指令后立即清零 `rx_cmd`，并开始非阻塞抓取动作

### 3.3 状态机时序（STM32 端）

```
视觉自主模式下：
  rx_cmd = 0x01 ──► Beep(1声) ──► 齿轮抓取序列（8步，约7秒）──► 复位
  rx_cmd = 0x02 ──► Beep(2声) ──► 螺母抓取序列（8步，约7秒）──► 复位
```

- 抓取过程中 **忽略新指令**（`vision_busy` 标志保护）
- 可随时按 **SELECT + START** 切回遥控模式（强制中断抓取）

## 5. 常见问题排查

### Q1: STM32 没反应（蜂鸣器不响）

1. 检查接线：TX/RX 是否交叉（ESP32-TX → STM32-RX）
2. 检查共地：GND 必须相连
3. 检查波特率：双方是否都为 115200
4. 检查 STM32 模式：是否已进入视觉自主模式（SELECT+START 响 3 声）
5. 用测试版 `esp32_test_sender.ino` 排除摄像头因素

### Q2: 识别准确率太低

- `esp32_stm32_direct.ino` 中的 `detectObject()` 使用的是**占位逻辑**（左右亮度差）
- 需根据齿轮/螺母的**真实颜色、形状、位置**重写识别算法
- 若 ESP32 算力不足，建议换用 **K210** 或 **ESP32-S3** 等带 AI 加速的模块

### Q3: 连续重复触发

- 已做 1.5s 软件防抖（`SEND_COOLDOWN_MS`）
- 若仍重复，可在 STM32 端增加硬件消抖，或提高识别阈值
